### 2. Evaluation Script
Five separate tasks were observed in total including:
1. Search for the newest post
2. Rating / voting a post
3. Add comments
4. Rating / voting a comment
5. Re-share post on users timeline

### 4. Feedback and Incidence Analysis
#### OBSERVATION 1 - Home Page
![Prototype Screen 1](/images/Observation1.png)

 - **Reference**: Participant 1 [line 22], Participant 2 [line 32]
 - **Feedback**: Negative feedback
 - **Incidence**: Participant 1 hard to recognize the function of "Sort By" button
 - **Reason**: There's no signifier that indicates the User is on their timeline page. The timeline page is so lack of content. Too much free space.
 - **Resolution**:
    - Highlighting the current page on Header
    - Add navigation that contains the "Sort By" dropdown menu
    - Increase the post container width

 
#### OBSERVATION 2 - Rate / Vote Button
![Prototype Screen 2](/images/Observation2.png)

 - **Reference**: Participant 1 [line 24], Participant 2 [line 28]
 - **Feedback**: Negative feedback
 - **Incidence**: -
 - **Reason**: The Participant is not familiar with Up-Rate and Down-Rate button for like or dislike a post. They prefer Thumbs or Love button for like some post. 
 - **Resolution**: I think that Up-Rate or Vote is way better than like or dislike only if people understand the differences. Nobody cares if you like or dislike a post. They care if a post is worth to read. Thatâ€™s what upvotes and downvotes are supposed to indicate. So I will not change the up-rate and down-rate button.

    - Change the color of hit number of rate

### Design Rationale
I have learnt from the contextual inquiry that not all users understand with our design. So we have to understand users. Every litle detail like highliting, color selection, and typography are important to attract the users. Sometimes users are not familiar with our feature (like up rate and down rate feature). As much as possible we should make a new design that's better without removing the characteristics of the website. By doing usability evaluation, again and again, we can see which problems users have and what they understand straight away. We might also see approaches or possible solution which is better.
